## 1. 服务类目 (categories)

**核心类目（一级，最多 5 个，资质门槛最低的优先）**：

| 序号 | 一级类目 | 二级类目 | 选它的理由 | 需要的资质 |
| --- | --- | --- | --- | --- |
| 1 | **工具** | **效率** | 留学日程/活动聚合 — 业务核心 | 无（基础类目） |
| 2 | **教育** | **校园助手** | 校园场景天然契合 | 无（基础类目） |
| 3 | **社交** | **社区/论坛** | 搭子匹配本质是社交 | 需要「社交-社区/论坛」类目（首次需备案） |

**首次备案流程**（社交类目）：
1. 登录 mp.weixin.qq.com → 设置 → 基本设置 → 服务类目 → 添加
2. 选「社交 → 社区/论坛」→ 提交「《社交-社区/论坛》类目申请函」
3. 申请函模板见 [§1.1 申请函模板](#11-申请函模板)
4. 审核周期 3-7 个工作日；未通过可重提

> ⚠️ **不要选** 「生活服务 → 社交交友」「娱乐 → 直播」等敏感类目。
> Pairhub 现阶段没有这些功能，误选会被打回 + 拉长审核周期。

### 1.1 申请函模板

```
致：腾讯微信开放平台审核组

我方申请贵平台「社交 - 社区/论坛」类目，应用于「Pairhub」小程序
（appid: wx0000000000000000）。本小程序核心功能为高校留学生活动搭子匹配
（自习、运动、桌游、开黑等），不涉及以下敏感场景：
  - 同城约会 / 陌生人 1v1 配对
  - 婚恋 / 相亲
  - 直播打赏 / 虚拟商品交易
  - 宗教 / 政治话题社区

用户真实身份：所有用户通过 WeChat / Apple / Google 三方 OAuth 登录，
手机号仅在用户主动授权后用于「同校匹配」功能。
内容审核：用户发布内容（活动标题/描述、评价）接入微信官方
`msg_sec_check` API 实时审核（issue #26 已落地），违规内容直接拦截。
举报机制：活动详情页与个人主页均设有「举报」入口，运营 24h 内响应。
未成年人保护：注册时强制勾选「我已年满 18 周岁」。

随函附：营业执照、ICP 备案（粤ICP备XXXXXXXX号）、运营负责人身份证。

申请单位：XXX 科技有限公司
日期：2026-XX-XX
```

## 2. 服务协议 + 隐私协议

**两套文档各两种语言**（简中 + 英文，因为 Pairhub 面向海外留学生）：

| 文档 | 路径 | 语言 |

| 服务协议 | [`legal/terms-of-service.zh.md`](./legal/terms-of-service.zh.md) | 简体中文 |

| 隐私协议 | [`legal/privacy-policy.en.md`](./legal/privacy-policy.en.md) | English |

**微信后台填写位置**：
1. mp.weixin.qq.com → 设置 → 基本设置 → **服务条款**（仅服务协议 URL）
2. mp.weixin.qq.com → 设置 → 基本设置 → **隐私协议**（仅隐私政策 URL）
3. 站内展示：登录页底部 + 个人中心「设置」页有外链（issue #33 落地时实现）

**URL 形态**（建议）：
- 短期：直接挂在 GitHub Pages / Vercel，URL 形如 `https://Pairhub.app/legal/terms-of-service.zh`
- 中期：迁移到 `https://Pairhub.app/legal/...`（自有域名 + 静态托管）

## 3. 用户隐私保护指引设置

微信后台 → 设置 → 基本设置 → **用户隐私保护指引**：

- ✅ 收集信息：手机号（仅同校匹配，可选）、位置（仅创建活动，可选）、微信 OpenID / Apple Sub / Google Sub
- ✅ 用途：账号体系、附近活动推荐、内容安全审核
- ✅ 存储期限：账号注销后 30 天全量清除（Redis token 立即清除 + DB 软删 + 7 天后 hard delete）
- ✅ 第三方共享：仅微信 OAuth 用于登录；不卖给任何第三方
- ❌ 拒绝授权的方案：游客模式可浏览首页 + 活动列表，创建/报名需登录

## 6. 常见被拒原因 + 规避

| 被拒原因 | 规避 |
| --- | --- |
| 「类目与功能不符」 | 类目功能描述要明确写「搭子活动匹配 / 校园活动聚合」 |
| 「未提供测试账号」 | 审核版本固定一个测试账号 (test@Pairhub / 密码 8位+字母数字) |
| 「诱导分享朋友圈」 | 不要写「分享到朋友圈得 XX 奖励」 |
| 「内容涉嫌违规」 | 后台接入微信 `msg_sec_check`（已落地）；评论先审后发 |
| 「登录后空白页」 | 真机跑通登录 → 首页 完整链路，不要只跑开发版 |
| 「类目资质不符」 | 第一次备案社交类目，备齐营业执照 + ICP + 身份证 |
| 「功能不完整」 | 不要先提交「只有首页」的版本；至少完成「登录 → 浏览 → 报名」闭环 |
