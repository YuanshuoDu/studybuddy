# Pairhub v1.0.0 — GA release notes

> **Released**: 2026-06-09
> **Tag**: `v1.0.0`
> **Commit**: `807fa50`
> **Platforms**: WeChat Mini Program (open) · iOS TestFlight (closed beta) · Android (Google Play internal test + 国内 APK direct)

Pairhub v1.0 is the first general-availability release of the
overseas-Chinese-student activity-matching app. Built over 12 weeks by
the in-repo M1-M3 plan, it ships **71 merged PRs** covering backend
APIs, 3 native clients, design system, marketing materials, ops
runbooks, and CI/CD.

## What's in v1.0

### Backend (Fastify + Prisma + PostgreSQL + Redis)

9 modules, ~3,800 lines of TypeScript:

- **auth** — social login (WeChat unionid / phone SMS / Apple ID),
  JWT access + refresh, 7-day token TTL, role + status embedded in
  the JWT payload
- **user** — me / profile / by-id / my-activities / avatar upload
- **activity** — list / create / detail / patch / cancel; geo "near
  me" with Haversine SQL; 6-state machine
  (PENDING_REVIEW → RECRUITING → FULL → STARTED → ENDED / REJECTED / CANCELED)
- **signup** — POST / DELETE / participants list; re-signup after
  cancel; PENDING approval for women-only / paid
- **review** — 1-5 star + tag cloud + 500-char comment; private by
  default
- **push** — register / list / unregister device tokens; composite
  unique (userId, channel, token); 404 on someone-else's token
  (anti-enumeration); noop dispatcher for M3 launch, real
  TPNS/FCM/APNs wire-up in v1.1
- **admin** — 6 operator endpoints (activity moderation, user
  management, dashboard metrics), RBAC, PENDING_REVIEW workflow
- **monitoring** — Prometheus `/metrics` (Bearer-gated), Sentry
  lazy-require + 脱敏, alert receiver (HMAC-verified, Feishu /
  DingTalk / generic webhook fan-out, 60s in-memory dedupe)
- **analytics** — 4 admin endpoints (funnel, retention,
  activity-volume, kpis); 30s Redis cache; view-stage in funnel is
  intentionally an approximation (real view-event log ships v1.1)

### Cross-cutting concerns

- **限流** — 6 endpoints, sliding window in Redis
- **微信内容安全** — title + description + cover image through
  `msg_sec_check`; fail-closed on 业务 errcode (87014), fail-open
  on network / 5xx (so dev/test aren't blocked)
- **content safety** — fail-open on network/5xx (dev/test
  tolerance), fail-closed on 业务 errcode 87014
- **audit logging** — every admin write goes to `UserReport` table
- **CORS** — strict allowlist (pairhub.app + localhost dev)

### iOS + Android (Flutter 3.24)

5 user screens each:

- Login (3-button: WeChat / phone / Apple)
- Activity list (filter chips + map toggle)
- Activity detail (cover + sign-up CTA + participants avatar stack)
- My profile (rating + tag cloud + review history)
- Map (real `MapboxMap` widget with markers + user tracking)

5 admin screens (issue #32 frontend):

- Gate (PIN-protected entry; safe-by-default for the user role)
- Dashboard (4-KPI snapshot)
- Activities (PENDING_REVIEW queue + Approve / Reject actions)
- Activity detail (full content + moderation note)
- Users (search + filter by status + ban / unban)

iOS TestFlight config in place; Android signed-release pipeline
ready (same `.jks` drives Google Play + 国内 APK direct so users can
upgrade seamlessly between the two channels).

## Known limitations (deferred to v1.1)

- **Real view-event log** for the funnel (currently approximated from
  the signup table). See `docs/v1.1-roadmap.md`.
- **Activity end-time auto-transition** (currently requires an admin
  PATCH; v1.1 ships a cron-like background job)
- **TPNS / FCM / APNs real wire-up** (push is noop for v1.0; SDK
  integration is in `app/`, backend stub is in
  `server/src/modules/push/`)
- **GCJ-02 offset** for miniprogram map (current offset is < 100m;
  acceptable for MVP). v1.1 ships full GCJ-02 ↔ WGS-84 conversion.
- **iOS App Store live listing** (TestFlight is up; App Store
  listing needs Apple developer enrollment + 1-2 weeks of review)
- **国内 Android markets** (we ship GitHub Release APK direct for
  v1.0; 华为/小米/OPPO/vivo/应用宝 上架 follows once we hit
  5k+ installs to justify the per-market审核 cost)
- **Soft-deletion** (User.banned is hard, no recovery; v1.1 adds
  soft-delete with 30-day undo window)
- **Real device farm screenshot capture** (issue #33 follow-up;
  planned v1.1)

## Upgrade notes

There is no v0.x to upgrade from — this is the first GA. The
data model is the one documented in
`docs/spec-v0.2.md`; Prisma migrations from `prisma migrate deploy`
land the schema in one step. Operators running v1.0 should:

1. `pnpm prisma migrate deploy`
2. `pnpm prisma:seed:large` (only against staging; refuses prod
   unless `ALLOW_SEED_IN_PROD=1`)
3. (operator) follow `docs/release/android-google-play.md` for
   Google Play rollout
4. (operator) follow `docs/release/android-cn-apk.md` for 国内 APK
   direct
5. (operator) set the 5 new env vars from `docs/ops/monitoring/`:
   `SENTRY_DSN`, `METRICS_TOKEN`, `ALERT_WEBHOOK_FEISHU`,
   `ALERT_WEBHOOK_DINGTALK`, `ALERT_RECEIVER_HMAC_SECRET`
